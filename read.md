# Outlook 自动化注册部署须知

## 部署后的第一优先级

1. 部署完成后，必须先适配并验证“自动代理注册 → 获取新订阅链接 → 更新 mihomo 订阅”的完整功能。
2. 必须确认已经成功获取到新的代理订阅链接，并写入：
   - `邮箱注册/mihomo_runtime/subscriptions.json`
3. 获取新订阅链接后，必须清理/淘汰旧的失效代理订阅链接，避免 mihomo 继续加载“订阅已过期”“服务器无法访问”等占位节点。
4. 必须重建 mihomo 配置并启动代理服务，确认代理订阅更新链路可持续自动运行。

## 关于本地直连注册

不要一刀切禁用本地直连注册。

很多服务器的本地直连 IP 可以正常注册 Outlook；只有部分服务器本地直连不行。判断标准应该来自实际注册结果和风控表现，而不是简单因为“出口 IP 等于服务器 IP”就直接判定失败。

正确策略：

- 如果服务器本地直连可注册，可以允许直连作为可用路径。
- 如果服务器本地直连持续 `account_blocked` 或表现不可用，再优先切换/刷新代理订阅。
- 不要在通用注册器里硬编码“本机 IP = 失败”。

## 代理订阅维护要求

自动代理功能必须做到：

1. 自动注册/申请新的代理账号。
2. 自动获取新的订阅链接。
3. 自动写入新的订阅链接。
4. 自动清理旧的、失效的订阅链接。
5. 自动重建 mihomo 配置。
6. 自动启动/重启 mihomo。
7. 注册前优先保证代理池是新鲜可用的，但不要因此禁用可用的本地直连注册。

## gcore 等裸机部署注意事项

裸机部署时不要假设 Zo 路径存在，例如 `/home/workspace/hidexx/hidexx-linux`。

必须按当前服务器实际路径适配：

- 项目根目录：通常是 `/home/ubuntu/Email-Register`
- hidexx CLI：可放在 `/home/ubuntu/hidexx/hidexx-linux`
- OCR 依赖：`tesseract-ocr`、`tesseract-ocr-eng`

如果自动代理注册依赖验证码识别或 CLI fallback，部署脚本必须先安装这些依赖并验证 CLI 可执行。
