# 代理系统修复报告

## 问题1：GLOBAL组设置错误导致代理未生效

### 症状
- 出口IP与直连IP相同
- 代理没有生效

### 根因
mihomo的GLOBAL组被错误设置为DIRECT，导致所有流量走直连而不是通过代理节点

### 修复
1. 在subscription_proxy.py启动时自动设置GLOBAL为AUTO
2. 在config.yaml中添加GLOBAL代理组

### 验证
```
修复前：代理出口 = 直连出口 = 66.97.122.3 ❌
修复后：代理出口 = 13.228.249.6 ✅
```
